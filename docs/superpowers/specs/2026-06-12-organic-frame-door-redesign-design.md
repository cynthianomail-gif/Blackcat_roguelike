# M8：有機剪影外框＋門重設計 — 設計規格

日期：2026-06-12
狀態：使用者已核准設計方向（有機剪影外框＋貓耳拱門＋光錐溢光＋荊棘鎖門）

## 背景與目標

M7 後整體質感提升，但房間的「前景黑色平面」仍是程式佔位美術：

- 牆/天花板/地板是筆直純黑矩形，視覺上讀成「UI 黑條」而非地形
- 地板每 40px 描一條白色格線，等距直線像試算表
- 門是發光金邊矩形＋紅色霓虹柵欄，與剪影美術方向衝突

問題共同根源：用「UI 的視覺語言」畫「遊戲世界的東西」。BADLAND 式美術要成立，
黑色前景必須有**有機的不規則輪廓**（Platform 已有垂鬚語言，牆與地板沒跟上）。

目標：不動任何遊戲邏輯、碰撞、平衡，純表現層重畫外框與門。

## 已拍板的方向決策

| 決策點 | 結論 | 理由 |
|---|---|---|
| 外框 | 有機剪影輪廓（土丘/凹凸/垂鬚/草叢/石頭）＋內緣暖色緣光 | 直邊是「像 UI」的元兇；緣光給逆光前景感 |
| 地板格線 | 直接刪除 | 等距直線最違和；層次改靠頂緣亮線＋稀疏剪影細節 |
| 門（開啟） | 貓耳拱門洞＋暖光光錐灑地＋飄浮光塵 | 門是「牆上挖的洞、光從裡面溢出」；貓耳呼應主題、成招牌記憶點 |
| 門（鎖閉） | 荊棘/交叉木板封口＋深處暗紅餘燼光 | 取代最俗的霓虹柵欄；剪影封口符合美術方向 |
| 實作方式 | 每房 seed 生成輪廓、烘焙離屏 canvas | 每幀直接貼圖，不重算幾何；零新美術素材 |

## 1. 有機剪影外框（FrameSilhouette）

新增 `js/render/FrameSilhouette.js`：負責為單一房間烘焙一張 900×506 的
離屏 canvas（透明底，只畫黑色前景框），房間切換時才重烘。

### 碰撞與視覺的關係（核心約束）

所有碰撞線（`FLOOR_Y`、`WALL_THICKNESS`、`CEILING_Y`）**完全不動**。
有機細節只能往「不影響站立判定」的方向長：

- **地板**：頂緣維持 `FLOOR_Y` 水平直線（站立面＝視覺面，貓不會浮空/陷地）。
  打破直線感的是「線上方的稀疏裝飾」：草叢、小石頭、小土丘（高度 ≤10px），
  以及沿頂緣的一條細暖色亮線（取代原本的白格線層次功能）。
  地板本體 `FLOOR_Y` 以下純黑 `#101014`，無任何格線。
- **側牆**：內緣以 seed 起伏，凸起一律**往遊戲區內長**（牆面在
  `WALL_THICKNESS` ~ `WALL_THICKNESS+8` 之間波動）。牆在渲染序 step 12
  （實體之後繪製＝蓋在實體上），玩家貼牆時被小凸起局部遮擋＝BADLAND 式前景遮擋，
  碰撞仍在 `WALL_THICKNESS`，不會有「隱形牆」感。
- **天花板**：同側牆邏輯（下緣往內長 ≤8px），加 2~3 條垂藤/垂鬚
  （細曲線，與 Platform 垂鬚同語言），長度 12~24px。

### 生成與烘焙

- 亂數：mulberry32 之類的簡單 seeded RNG，seed＝`floorNum * 1000 + gridPos.y * 16 + gridPos.x`，
  同一房每次進入輪廓一致。
- 輪廓線：沿邊以固定步長（約 40~60px）取點、隨機振幅、quadraticCurveTo 串起來。
- 裝飾密度：地板草叢/石頭約每 100~140px 一處（隨機抖動），**避開 S 門開口區**
  與玩家出生點附近。
- 緣光：沿地板頂緣與牆內緣描 1.5px 暖色細線 `rgba(255,225,160,0.2)` 左右，
  地板頂緣可稍亮（兼任站立面提示，承接原格線的功能）。
- 門開口：烘焙時依 `room.doors` 在對應位置**留洞**（描邊路徑直接繞過開口，
  或畫完後 `destination-out` 挖掉）。洞形＝貓耳拱（見下節），門的動態光效
  另由 `Door.draw` 每幀畫。

### 渲染接線

- `Room.drawFloor`（step 4，實體之前＝實體會蓋在地板裝飾上）：貼烘焙圖的
  「地板部分」＋平台（不變）。
- `Room.drawWalls`（step 12，實體之後＝最上層前景）：貼烘焙圖的「牆＋天花板部分」。
- 實作上烘焙兩張（floor 層 / walls 層）或同張分區裁切皆可，以簡單為準。
- `Renderer.js` 管線順序不變；`drawFallbackBackground` 佔位邏輯不動。

## 2. 門重設計（Door.draw 全面重寫）

門的觸發區 `rect`、開關邏輯、事件**全部不變**，只改 `draw`。

### 開口形狀：貓耳拱

- E/W 門：牆上挖出貓耳拱形洞（圓拱頂＋兩個耳朵尖），寬約 44px、
  高 `DOOR_H`（80px），底對齊 `FLOOR_Y`。形狀由 FrameSilhouette 烘焙時留洞。
- N 門：天花板上的不規則圓洞（洞緣有 2~3 個碎凸起），貓耳拱在天花板方向
  不易讀，改用洞形。
- S 門：地板上的不規則破洞（同 N 門語言），洞緣不規則。

### 開啟狀態（光從洞裡溢出來）

- 洞內填暖色亮面（接近 `#FDF3DC`，保留現有緩慢脈動 pulse）。
- **光錐**：自洞口向遊戲區灑出一道斜光錐（半透明暖色多邊形，
  E/W 門落在地板上、N 門向下灑、S 門向上透），落地處加一橢圓光池。
  光錐是「這是通道」的主要讀法，取代金邊框。
- **光塵**：洞口附近 3~5 粒微小光點緩慢飄浮（正弦漂移，無需粒子系統，
  在 Door.draw 內以時間函數直接畫）。
- 移除：金色描邊矩形、shadowBlur 發光邊框。

### 鎖閉狀態（剪影封口）

- 洞內改深暗色，深處一小團暗紅餘燼光（`rgba` 暗紅徑向，微弱脈動，
  亮度遠低於開啟狀態——「裡面有危險」的暗示）。
- 洞口被**交叉荊棘/木板剪影**封住：2 條粗黑圓頭斜線交叉＋4~6 根小刺，
  純黑 `#101014` 與前景同色。
- 清怪開門時荊棘消失（可直接消失，淡出為加分項不強求）。
- 移除：紅色霓虹柵欄、紅色發光描邊。

## 3. 驗收

1. 零邏輯更動：現有 smoke test 維持綠燈；門觸發、子彈撞牆、玩家 clamp 行為不變。
2. preview 逐項目視：六層樓各看一房（輪廓隨樓層/房間變化）、四個方位門的
   開啟/鎖閉狀態、戰鬥鎖門→清怪開門流程、S 門按下進入、貼牆與貼天花板時
   的前景遮擋觀感。
3. 截圖比對：同房改造前後對照，確認「UI 感」消失。
4. 效能：烘焙只在進房時發生，每幀新增成本僅 drawImage×2＋門光效，
   目測 FPS 無感。

## 不做的事（本次範圍外）

- 不動碰撞線、判定、房間生成、門邏輯
- 不產任何新 PNG 素材（全程 canvas 程序繪製）
- 背景圖、HUD、平台繪製不動
- 不做門的開關動畫（荊棘淡出為加分項）
- 主選單/轉場不在本次範圍
