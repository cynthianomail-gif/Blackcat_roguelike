# M9 設計規格:怪物行為強化＋居合大招＋底部土層帶＋排水柵 S 門

日期:2026-06-13
狀態:已與使用者逐段確認(視覺輔助選項+終端逐題核可)
前置:M8 有機剪影外框已完成(FrameSilhouette/DoorShapes 架構沿用)

## 1. 背景與目標

真人試玩回饋四個問題:
1. 部分怪物「飄在空中」違和;盤點後收斂為**麻雀保持距離懸停**死板(其餘飄浮怪——幽靈/燈籠/書魂等——使用者確認合理)
2. 多數怪行為單調,希望參考《以撒的結合》怪物行為原型增加深度
3. L 鍵 EX 能量彈不夠爽,想改近距離大招+黑白酷炫特效
4. 底部黑色帶 106px 比牆(60px)厚近一倍,像一條過高的 UI 列;S 門在其上缺「向下」語言

對應使用者決議:
- 怪物:**中度強化**——動 7 隻,其餘 11 隻不動
- L 大招:**B 居合爪連斬**(黑白漫畫分鏡+爪痕+位移)
- 底部:**C 混合案**——縮到 80px+土層斷面質感
- S 門:**B 排水柵口**(掰彎欄杆)+**樓層光色查表**

## 2. 範圍與非目標

**範圍**:`behave()` 層行為改寫、Player EX 釋放邏輯、Constants 平移、FrameSilhouette 地板層裝飾、Door S 門繪製。

**非目標**:
- 不動任何怪的 HP/傷害/移速數值(平衡已試玩校過)
- 不動碰撞箱、掉落表、房間編成權重
- 不動 Boss 行為
- 不重生成任何美術素材(特效全部程式繪製)
- 不動 E/W/N 門(M8 貓耳拱照舊)

## 3. 怪物行為強化(7 隻)

設計原則:**只改 `behave()`/`update()` 行為層**;每隻給「可被玩家學習的行為循環」(前搖→行動→破綻窗口);新增常數一律進檔頭常數區,嚴禁 magic number。

| # | 怪(樓層) | 現狀 | 新行為 | 以撒原型 |
|---|---|---|---|---|
| 1 | 麻雀 Sparrow(F1) | 懸停玩家側上方 200px+定時射擊 | **俯衝轟炸**:自由盤旋(不鎖玩家距離)→鎖定前搖閃白 24f→沿弧線俯衝穿過玩家當時位置→拉起回盤旋。**刪除射擊**(子彈/間隔常數一併移除) | 俯衝鳥 |
| 2 | 鴿子 Pigeon(F1) | 直線追擊貼臉磨 | **鐘擺式俯衝**:朝玩家俯衝,衝過後慣性拉起盪向另一側高處,再折返。不再原地貼身 | Swooper |
| 3 | 蟑螂 Cockroach(F2) | 地板 Z 字爬 | **全畫面爬行**:貼表面沿「地板→牆→天花板」環繞爬行(剪影隨表面旋轉);爬到玩家頭頂上方 ±30px 時**落下突襲**(自由落體+落地小停頓),落地後續爬 | 蜘蛛/爬牆者 |
| 4 | 老鼠 Mouse(F2) | 原地啃 3 秒摧毀道具 | **叼著跑**:接觸道具 60f 後叼起逃跑(遠離玩家方向,速度 ×1.3);擊殺→道具掉回腳下;叼走 480f(8 秒)未被擊殺→鑽進牆縫,道具真消失 | 小偷 Greed |
| 5 | 流浪狗 StrayDog(F3) | 直接衝撞無預警 | 加**吠叫前搖 20f**(壓低身體+腳下揚塵粒子)再鎖方向衝撞;**撞牆自暈 30f**(現為立即喘息 50f,改成暈眩才有「誘導撞牆」打法) | Charger |
| 6 | 廚師機器人 ChefBot(F2 精英) | 360° 油滴散射純站樁 | 油滴**落地留油漬區 150f**:玩家踩到摩擦力驟降(慣性增大、操控滑移)。複用 InkZone 架構(改效果為打滑,不是減速) | 地形污染者 |
| 7 | 燈籠鬼 Lantern(F6) | 懸浮追蹤+死亡四向彈 | 距玩家 120px 內**燈火轉紅+加速 ×2.2 自爆衝刺**,接觸或 60f 後自爆(=觸發現有死亡四向彈)。遠程擊殺=安全;放近身=吃衝刺 | 自爆怪 |

**不動的 11 隻**:胖信鴿、垃圾桶怪、大鼠(已有鑽地循環)、BoxBot、精英鼠王(已有衝刺)、書魂、墨水章魚、圓規怪、靈狐(已有閃現循環)、石狛犬、幽靈。

蟑螂爬牆實作備註:以「表面參數 t」沿房間內緣矩形(地板/右牆/天花板/左牆)推進,繪製時依所在表面旋轉 sprite(90°/180°/270°);碰撞箱仍用 AABB(貼表面時箱體中心在表面內側)。落下突襲期間恢復普通重力位移。M8 牆內緣有 ≤8px 有機凸起,但碰撞線是直的——爬行沿碰撞線即可,視覺誤差可忽略。

## 4. L 大招:居合爪連斬(取代 EX 能量彈)

集氣不變:命中 +20、上限 100、滿格按 L(`input.exPressed`)。釋放全換:

**四階段(合計約 36f)**:
1. **啟動 6f**:玩家輸入鎖定、擺前傾架式;全屏黑白反轉(在 Renderer 最上層畫 `globalCompositeOperation = "difference"` 白矩形,零成本)
2. **衝刺 10f**:朝 `facing` 方向位移 `EX_SLASH_RANGE 280px`(每幀 28px,撞牆提前停,沿用既有牆 clamp);全程無敵;路徑走廊(玩家中心線上下各 `EX_SLASH_CORRIDOR_H/2 = 45px`)內敵人標記+定格;留 3 道白色爪痕線+Dash 殘影(複用 `afterimages`)
3. **定格 20f**:全屏黑白斜紋分鏡遮罩(repeating 斜紋+difference);被標記敵人白閃顫抖(複用 `hurtFrames` 視覺,不觸發 AI 中斷邏輯——直接設新 `slashMarkFrames`)
4. **結算**:標記敵人同時吃 `totalDamage × EX_SLASH_DMG_MULT(6)` +擊退(`knockX = ±10`),白色爆裂粒子(複用死亡粒子發射器)

規則:
- **Boss**:吃全傷害,但**免疫定格演出**(不設 `slashMarkFrames`,結算時直接扣血)——不破壞 Boss 行為機與 Phase 切換
- 玩家結算後再給 10f 無敵緩衝(落點可能在怪堆裡)
- 衝刺不觸發門(門觸發靠按鍵/區域邏輯不變),不穿牆
- 移除舊 EX 彈路徑:`shootVolley(true)`/`spawnBullet isEX` 分支與 `EX_BULLET_*` 常數退役(保留 `EX_ENERGY_*`)
- 音效:EX 觸發音+Dash 音疊加,不新增素材;`playerShootEX` 事件名沿用(AudioEvents 不用改)

新常數:`EX_SLASH_RANGE 280 / EX_SLASH_DMG_MULT 6 / EX_SLASH_CORRIDOR_H 90 / EX_SLASH_STARTUP 6 / EX_SLASH_DASH 10 / EX_SLASH_FREEZE 20`。

數值錨點:舊 EX 彈單體 ×5 遠程零風險;新居合多體 ×6 但須貼近+直線命中,風險換報酬。

## 5. 底部黑帶:106 → 80px+土層斷面質感

**常數平移(+26)**:
- `FLOOR_Y 400 → 426`(底帶 = 506−426 = 80px)
- `PLATFORM_TIER1_Y 280 → 306`、`PLATFORM_TIER2_Y 170 → 196`
- `ROOM_H`/`ROOM_W` 經 grep 確認**從未被使用**(死常數):一併更新為 426 並補註,或直接刪除(實作時擇一,傾向刪除)

跳躍物理驗算(跳高 141px,與現版餘量完全相同):
- 地板(426)起跳腳底達 285 ≤ TIER1 306 ✓(餘 21px)
- TIER1(306)起跳腳底達 165 ≤ TIER2 196 ✓(餘 31px)
- TIER2(196)站立頭頂 152,起跳頭頂達 11 ≤ 北門觸發 62 ✓

風險掃描:實作時 grep 寫死 `400`/`106` 的地方(已知 Door.rectFor 用常數、敵人 spawn 用常數,自動適應;Camera/背景 cover-fit 不受影響)。

**土層斷面質感**(烘進 `FrameSilhouette.bakeFloor`,零每幀成本):
- 1–2 條極淡地層橫線(`rgba(255,232,170,.05–.07)`)
- 2–4 顆埋藏石塊剪影(`#1b1b22`,比底色 `#101014` 微亮)
- 1–2 根從地表往下的根鬚微光線
- 底緣 1px 紫微光(`rgba(120,80,200,.35)`)
- 全部用既有 seeded RNG,序列消耗與門無關(沿用 M8-2 修正原則)

## 6. S 門:排水柵口+樓層光色

門洞挖法照舊(`bakeFloor` punch);柵欄與光效畫在 `Door.draw` 執行期(與貓耳拱同架構):

- **柵欄**:門洞內 5 根直欄剪影(`#101014`),欄距均分 60px 門寬
- **開門**:中間 2 根欄杆掰彎(一根 +14°、一根 −12°,開出貓縫);柵縫透 **3 束呼吸光柱**往上打 ~50px(錯相位 sin 呼吸,參考 M8 門光塵實作)+少量上飄光塵
- **鎖門**(戰鬥中):欄杆全直、無光柱——清怪瞬間「欄杆掰彎+光亮起」即開門演出(狀態切換用既有 `door.isOpen`)
- **光色查表**(光柱+光塵著色,暗示下一層):

```js
const FLOOR_LIGHT_COLORS = {
  2: "#ffd75e",  // 廚房暖黃
  3: "#6ebeff",  // 雨夜藍
  4: "#e8a13c",  // 倉庫琥珀
  5: "#ffc878",  // 燭光金
  6: "#dce8f0",  // 月白
  7: "#ff5e3a",  // 終點緋紅
};
```

索引 = 目前樓層 +1(S 門必通下層)。

## 7. 驗收標準

1. **怪物**:7 隻新行為逐隻實測(F1/F2/F3/F6 入房觀察完整行為循環);麻雀不再鎖距離懸停;蟑螂可爬全畫面+頭頂落下;老鼠叼道具跑、擊殺掉回、逾時消失;其餘 11 隻行為與改前一致
2. **大招**:集滿按 L 走完四階段;走廊內多敵同時結算 ×6;Boss 吃傷不定格;撞牆提前停;結算後無敵緩衝生效;EX 條歸零重新集氣
3. **底帶**:F1–F7 全樓層巡檢,黑帶 80px+土層質感;三層平台跳躍鏈與北門可達性實測;無寫死 400 的殘留錯位
4. **S 門**:鎖門全黑直欄→清怪掰彎亮光;光色隨樓層查表正確;與 M8 烘焙層無接縫錯位
5. **零邏輯回歸**:門觸發/子彈撞牆/拾取/商店價格等既有流程煙霧測試零 console error
6. **效能**:土層裝飾全烘焙;大招特效僅釋放期間 ~36f 有額外繪製
